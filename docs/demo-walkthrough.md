# AlignOps Demo Walkthrough

## Positioning

AlignOps is an enterprise execution governance operating system. Do not pitch it as a goal tracker. Pitch it as the control layer that keeps employee goals, manager approvals, quarterly check-ins, escalation, audit, and governance reporting aligned at scale.

## 7-Minute Flow

1. Log in as `nisha.iyer@alignops.local`.
2. Show the Goal Cockpit: governance health, KPI confidence, quarterly timeline, forecast, and goal-level SMART criteria.
3. Switch to `aman.gupta@alignops.local` if available, or show the manager view for Aman. Point out duplicate KPI, unrealistic target, and likely-to-miss deterministic warnings.
4. Log in as `rohan.mehta@alignops.local`.
5. Open Manager Team Queue. Show approval queue prioritization, execution health engine, needs attention queue, performance distribution, high-risk goals, escalation radar, and Teams/email notification preview.
6. Return one risky goal sheet with a comment, then approve a clean one if available.
7. Log in as `aditi.rao@alignops.local`.
8. Show Admin Control Tower: org execution health, bottleneck insights, department heatmap, lifecycle matrix, escalation trends, audit pulse, operational metrics, and CSV export.
9. Open Reports. Download the governance CSV and show thrust-area analysis, UoM distribution, QoQ trend, and lifecycle distribution.
10. Open Governance. Show the unlock intervention center and explain the auditable relock flow.

## Winning Moment

The strongest story is:

Employee goal quality problem -> manager receives an enterprise-style notification -> manager reviews with risk context -> admin sees the governance impact and can export the report.

That connects Phase 1 and Phase 2 requirements into one operating narrative.
